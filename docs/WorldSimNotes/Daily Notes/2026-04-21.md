[[Main Devlog Note]]
What did I work on?
- Got some more structure made for all this shit
	- We have a faction with a supreme leader (name is wip)
	- Each faction has several departments
		- These departments control the construction, upgrade, repair, and status of a certain building
		- Those buildings can be leveled by spending more resources
		- The buildings level determines the health (time till repair needed), output, required input, and efficiency
		- Certain levels unlock new abilities for the building
		- Research is required to unlock building levels
	- Just now realizing that an agent could get stuck in a loop of trying to build a building level that it does not have access to
		- Instead I will have the command just be build 10% or build 20%. This will direct the department to build new buildings by using up to a certain percentage of available resources
		- The percentage should be governed by the faction leader or the department lead if they are a colony lead.
		- If the percentage exceeds the allowable then the trust between the subordinate and the leader will lower
		- When trust gets low enough the leader can have the lower executed and a new agent with different weights and biases will be put in their place
		- At the same time the leader will provide feedback which can help the agent to learn in real time
	- Learning to use IntEnum, dataclass, and config structures to make things easier in the long run if I add more resources, buildings or levels.
		- Reference the Claude conversation to read about IntEnums and dataclass
- First goal will be to have a faction utilize the resources within a system
	- Sim will end after a certain number of ticks or when all resources are utilized on the planets
		- hoping this works because it will maximize the episode length or fail to generate a self sufficient faction which reduces score
	- Score always starts at 0 and builds, making it fairly easy for the model to learn what actions are better than others.
		- Im expecting some serious exploits in the beginning so I will need to tailor the reward function to make those exploits not as great
	- to reduce the impact of exploits early I can use a logarithmic relationship divided by the max resources in the system.
		- This will allow for an increasing score up to 100% utilization and then diminishing returns after 100% utilization
			- piecewise function might be better than logarithmic so the utilization up to 100% is linear.
		- For resources that are non renewable like minerals, energy, and rare materials I can scale their reward to be much larger than things like organics, power, and building materials which can be infinite
			- Organics are infinite so that food can be provided for the population. If a faction focuses on food then there will be way more food than people need so they would maximize score
			- power is also infinite and will be worth more than organics
			- building materials can be created from organics so they are also infinite but requires heavy amounts of research
		- This should make it more rewarding for the model to pursue minerals and energy in early game and then rare materials in mid game, then for the end game it should max out the other infinite materials to max out score before the time limit runs out.
		- Defense score can also be a bonus to the score but isn't super useful right now.
			- I dont expect defense to be in the "meta" for this peaceful sim, but as soon as factions can attack each other or there are enemies on the maps defense will become much more useful
			- I should add additional rewards for killing enemies with a bonus if a conflict is won and a territory gained
		- Normalizing the resources by the amount available should make it harder for resource abundant systems to gain rewards much quicker than the resource scarce systems
	- This will take a while to implement especially when it comes to the agents because they will rely on each other fairly heavily.
